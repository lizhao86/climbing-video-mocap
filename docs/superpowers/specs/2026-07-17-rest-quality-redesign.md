# 休息点质量判据重做（B 阶段）— 设计定稿

2026-07-17。起因：IMG_6411 首次触发「休息点质量」即误判满分（详见 PLAN.md §8）。
老板已定方向：**这项指标评价「已发生的停顿休得好不好」，不做「错过的休息机会」判断**
（后者是 A 阶段，做完 B 后单独一轮，需要「手脱开甩手」检测）。

## 病根（三个，互相独立）

1. **「直臂 +15」既是准入门槛又是加分项**：`is_rest` 要求 straight 才成立，quality 又给
   straight 加 15 分 → 凡被判休息必拿这 15 分，是假分辨率。60+15+10+15=100 的满分是
   结构必然，不是巧合。
2. **`elbow_open = max(左,右)` 只看张得更开的那条臂**：IMG_6411 右臂 155° 直、左臂 137°
   弯，只报 155。弯着的那条臂回不了血，质量分却完全无视。
3. **`FOOT_HIGH` 是单边阈值 `gap < 0.80`，缺下界**：IMG_6411 的 gap=-0.21（脚踝高过髋）
   照样通过并打出「重心压脚(+)」。脚高过髋只发生在陡壁挂姿，那是核心绷紧，与「把重量
   压给脚」相反——判据默认了墙是竖直的。

## 修法（全部落在 climb_report_v2.py，旋钮区可调）

| # | 改动 | 旋钮 |
|---|---|---|
| 1 | 基础分 60→75，删除 Q_STRAIGHT(+15) | Q_BASE=75 |
| 2 | 压脚加分改区间 `GAP_PRESS_LO ≤ gap < FOOT_HIGH`；gap<0 另扣 10 分，理由「脚高过髋，重量压不到脚上」 | GAP_PRESS_LO=0.0，Q_STEEP_HANG=-10 |
| 3 | 另一条**可见**的臂（过两道体检）中位肘角 <150° → 扣 20；不可见不扣（存疑从宽） | Q_OTHER_BENT=-20 |

分值空间：素休息 75；满分 75+10+15=100；IMG_6411 预期 75+15−20−10=**60**。
准入门槛不变（至少一条臂直 + 低速 + ≥2s）——方案 2 已定：IMG_6411 那类停顿**仍算休息**，
只是质量分低，这正是「评价已发生的停顿」的语义。

## 依据

- 教材（知识库）：外側掛旗法「把腰**大幅沉下来**休息」(p.82-84)、鎖膝法「让手放松休息」
  (p.110)——教材口径的休息 = 腰沉下去、重量离开手。脚高过髋与「沉腰」相反。
- 数据：六条素材全部 rest 记录只有 IMG_6411 一条（gap=-0.21 的误判），**没有正例样本**
  → gap 下界不能靠数据拟合，只能取物理硬边界 0（脚高过髋则重量物理上压不到脚，
  锁膝/挂钩是例外但教材说「机会不多」且 rule_weak，不为它弯曲主判据）。
  GAP_PRESS_LO 留成旋钮，等六盘水（唯一含真休息的素材）取数后用真样本回来精调。

## JSON 契约变化（metrics_v2.json → rests.items[]）

- 新增：`left_elbow_med` / `right_elbow_med`（逐臂中位肘角，不可见为 null）、
  `other_arm_bent`（bool|null）、`steep_hang`（bool, gap<0）。
- `elbow_open_med` 保留（向后兼容），`foot_high` 语义修正为区间判定结果。
- 展示层 climb_report_card.py 只读 reasons 字符串，无需改动（验证时确认）。

## 验证

1. 重跑六条素材：IMG_6411 质量 100→60，reasons 不再含「重心压脚」，新增「另一条胳膊
   弯着」「脚高过髋」；其余五条 rests 仍为空、其他指标零变化（diff metrics_v2.json）。
2. 报告卡重新生成后人工看 IMG_6411 的休息条目文案。
