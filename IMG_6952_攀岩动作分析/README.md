## 关键结论

- 净上升约 5.0 身长，攀爬效率 22.1%（横向 17.4 略大于纵向 12.6）。
- **脚法均衡**：左脚 11 次、右脚 10 次移动——比上一条线（右脚仅 3 次）明显改善。
- 全程 33 个 move，流畅度评分 52.8/100（中等偏上）。
- 静止/锁定占比 35%。
- 自动识别 5 个吃力点（crux）：2.4 / 8.6 / 29.5 / 34.0 / 36.6 秒；后三个集中在顶部区段（收顶/mantle 阶段，垂直进展不大但吃力）。

## 复跑 / 分析新视频

```bash
# 1) 提取骨架
CLIMB_MAX_W=960 CLIMB_COMPLEXITY=2 python3 脚本/climb_pose.py <视频.mov> <输出目录>
# 2) 算指标 + 出图 + 截crux + 生成报告卡（一步到位）
python3 脚本/climb_analyze_report.py --dir <CSV目录> --base <文件名前缀> \
    --annotated <标注视频mp4> --out <报告输出目录> --route "V2 抱石·绿线"
```
依赖：`mediapipe opencv-python numpy matplotlib` + 系统 `ffmpeg`。

## 数据口径

单摄像头 + 相机静止假设。长度/速度以「身长」为单位（尺度无关），反映相对趋势与节奏，非实验室级绝对测量；单目对深度方向不敏感。需要真实米制/深度/可导 Blender 的三维数据，建议多机位或 GVHMR 等世界坐标方案。
